Git submodule

發表於 2021-03-07

需求：

  • Submodule path: <folder path>
  • SubRepo url: <repo url>

Add Submodule

  1. add submodule

  2. 確認 working tree 狀態

    • .gitmodules: 記錄 module 資料夾路徑及 repo url
    • themes/next/source/lib/pjax: module 資料夾路徑

  3. 記錄 repository 的變更

Pull Submodule

如果 SubRepo 更新，可以在 git status 看到提示，但在 SuperRepo 執行 Pull 並不會更新。

SubRepo 更新：

  • –recursive: 如果 submodule 中有其它的 submodule，將會一併進行更新。
  • –remote: 基於遠端資訊更新，而非本機上的HASH資訊。

Push Submodule

使用方式同 SuperRepo。

Delete Submodule

  1. 將 submodule 取消註冊

  2. 刪除 submodule 在 .git 的資訊

  3. 移除 submodule 資料夾

Git submodule and subtree

發表於 2021-03-07

Submodule vs Subtree

Submodule 與 Subtree 兩個都是可以將 SubRepo 加入 SuperRepo 的解決方案，但解決方法與使用上都是不同的。

  • Submodule 像是指標的形式，將 SubRepo 的 hash 記錄在 SuperRepo。
  • Subtree 以副本形式，將 SubRepo 複製一份到 SuperRepo。
Submodule Subtree
Cost 僅有 .gitmodule 同 SubRepo 大小與空間
Clone SuperRepo 需多步驟 原指令
Push to SubRepo 簡單，視為獨立的 repo 可以直接 Push 麻煩，需要比對 commit 版本
Pull to SubRepo 麻煩，須另外執行指令 簡單，直接 Pull SuperRepo 即可
簡單描述 一個 Repo 中另一個 Repo 與原 Repo 結合，視為子目錄即可
